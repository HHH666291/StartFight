# StartFight 开发路线

路线图只说明推荐顺序，不代表当前重构新增了这些玩法。

## Phase 1：当前已完成

- 玩家移动
- 玩家攻击
- 敌人生成
- 敌人追踪
- 角色血量
- 经验升级
- HUD 与身体血条
- 攻击扇形视觉
- 脚本职责与目录收敛

## Phase 2：下一步建议

- 敌人攻击玩家：启用并完善 `EnemyContactDamage`
- 玩家死亡：连接 `CharacterDeath` 与 `GameManager.GameOver`
- 游戏结束：增加最小结束界面
- 攻击冷却：在玩家攻击入口增加简单计时
- 伤害数字：把伤害结果接到 `DamageNumberView`

## Phase 3：中期

- 新敌人类型
- 技能
- 掉落
- 波次
- 暂停菜单

## Phase 4：后期

- 存档
- 装备
- 关卡
- 音效
- 更完整的 UI

每个阶段都应先完成可玩的最小闭环，再决定是否需要新抽象；不要提前建立完整技能、掉落或 UI 框架。
